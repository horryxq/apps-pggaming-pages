# apps-pggaming-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不依赖于特定的域名或网站，可作为独立资源访问。

## 项目简介

该仓库主要服务于前端静态页面的存储与分发。所有页面均为独立的 HTML 文件，无需后端支持，可直接通过浏览器打开或部署至任意静态托管服务。

## 目录说明

```
├── index.html          # 入口页面（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md
```

- `pages/`：存放所有独立页面文件，按需命名。
- `assets/`：存放页面共享的静态资源，便于统一管理。

## 页面归档说明

- 所有页面均为纯前端实现，无后端依赖。
- 页面间相互独立，可单独访问。
- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并确保引用资源的路径正确。
- 不建议在页面中硬编码特定域名或外部链接，以保持通用性。

## 维护说明

- 欢迎提交 Pull Request 更新或新增页面。
- 请保持目录结构清晰，避免重复资源。
- 如需删除或废弃页面，建议在提交说明中注明原因。
- 本仓库不提供任何形式的动态服务或数据接口。
